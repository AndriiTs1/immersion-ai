# IntegrationAI

## Speak First Method — V1 Blueprint

Цель V1: пользователь (на старте — сам разработчик) способен через 30 дней поддерживать
5–10-минутный разговор на итальянском о собственной жизни без постоянного перехода
на родной язык.

Никакого «магического секрета» в основе нет. Эффективность строится на сочетании
четырёх проверяемых принципов, которые формализованы ниже в виде продуктовых модулей.

---

## Принципы

### 1. Коммуникация важнее грамматики

На ранних этапах цель — не знание правил, а способность поддержать разговор.
Пользователь должен как можно раньше начать представляться, рассказывать о себе,
задавать вопросы и понимать типовые ответы. Грамматика появляется постепенно —
как объяснение уже знакомых речевых шаблонов.

### 2. Повторение разговорных паттернов

Основа обучения — наиболее частотные конструкции языка. Цикл:

Слушаю → Повторяю → Произношу самостоятельно → Использую в диалоге

Цель — сократить внутренний перевод с родного языка и довести базовые конструкции
до автоматизма.

### 3. Личная релевантность

Фразы, связанные с реальной жизнью пользователя, запоминаются значительно быстрее,
чем абстрактные учебные примеры («в отеле есть бассейн», «погода сегодня хорошая»).
Вместо этого — «я разработчик», «я живу в Лугано», «сегодня у меня встреча»,
«я работаю с искусственным интеллектом». Язык становится инструментом для выражения
собственных мыслей, а не набором учебных упражнений.

### 4. Разговор раньше идеальности

Цель первого этапа — свободное общение, а не безошибочная речь. Пользователь должен
научиться говорить без длительных пауз, не бояться ошибок, поддерживать разговор и
понимать собеседника. Произношение и грамматическая точность улучшаются постепенно
через практику.

---

## Доменная модель

Вся система крутится вокруг одного объекта — `Phrase`. Каждый модуль (Phrase Engine,
Shadowing, Speaking Lab, AI Partner) использует или порождает `Phrase`, поэтому он
зафиксирован как главная сущность с самого начала.

```ts
type Phrase = {
  id: string;
  it: string;
  ru: string;
  context: string;
  difficulty: number; // 1–5, для подбора следующей фразы
  tags: string[]; // например: ["work", "self-intro", "daily"]
};
```

`difficulty` и `tags` не используются активно в V1, но без них пришлось бы позже
переписывать структуру данных и весь Phrase Engine. Дешевле заложить сейчас.

---

## Языковая абстракция (заложено сейчас, используется в V3)

V1 работает только с итальянским — никакой логики выбора языка не строим. Но чтобы
не переписывать Phrase Engine при добавлении новых языков в V3, формат данных сразу
проектируется как набор, а не как единственный хардкоженный список:

```ts
type LanguagePack = {
  code: string; // "it", "en", "de", "es"
  name: string;
  phrases: Phrase[];
  voice: string; // голос для TTS (BCP-47, например "it-IT")
};
```

В V1 `lib/phrases.ts` — это просто один `LanguagePack` с `code: "it"`, без
переключателя языков в UI. Добавление нового языка в V3 — это новый файл с данными,
а не новая архитектура.

## Архитектура продукта

### 1. Phrase Engine (V1)

Базовое аудио-лингвальное ядро.

1. Система произносит фразу (TTS).
2. Пользователь повторяет вслух.
3. STT распознаёт речь.
4. Система сравнивает результат с эталоном.
5. Пользователь повторяет до уверенного воспроизведения.

Технически: чистый клиент (браузер), без бэкенда — `Web Speech API` (STT + TTS),
данные фраз в `lib/phrases.ts`, прогресс в `localStorage`.

### 2. Shadowing (V1.1)

Тренировка ритма и интонации.

1. Проигрывается фраза без текста.
2. Пользователь повторяет вслед за голосом.
3. Только после ответа отображается оригинальный текст.

### 3. Speaking Lab (V1.2)

Переход от повторения к самостоятельному говорению. Пользователь получает тему
(работа, семья, планы, город, повседневная жизнь) и говорит 30–60 секунд без
подсказок. Система сохраняет аудио, строит транскрипт, показывает ошибки и
улучшения.

### 4. AI Partner (V2)

Полноценный разговорный собеседник: свободный диалог, ролевые сценарии (заказ
кофе, аренда квартиры, знакомство, рабочая встреча, разговор с другом), адаптация
сложности, обратная связь после разговора.

Технически: требует серверного слоя — Next.js API routes (`/api/mission`,
`/api/feedback`) + Claude API с серверным ключом.

---

## Не входит в V1

- Авторизация
- Регистрация пользователей
- База данных
- Поддержка нескольких языков (UI-переключатель)
- AI-диалоги (это V2)
- Мобильное приложение
- Система достижений
- Социальные функции

Любая фича из этого списка, всплывающая в процессе разработки V1 — повод записать
её в backlog, а не реализовывать сейчас.

---

## Метрики успеха

Помимо цели «через 30 дней» нужны ежедневные ориентиры, чтобы понимать, работает
ли метод, а не ждать месяц вслепую:

- Фраз изучено: 200+
- Процент успешного повторения: >80%
- Длина монолога (Speaking Lab): 60 секунд
- Дней подряд (streak): 30
- Время практики в день: 10–15 минут
